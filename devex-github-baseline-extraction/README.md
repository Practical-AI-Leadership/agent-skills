# GitHub DevEx Baseline Extraction

Extract 9 baseline developer experience metrics (M1-M9) from a GitHub repository: PR cycle time, PR size, time to first review, review comment depth, commits per PR, rejection rate, revert rate, hotfix frequency, and commit distribution by author. Produces a one-page quantitative baseline from real GitHub PR data.

## Problem

Teams argue about slow delivery without measurements. This skill converts anecdotal frustration into 9 specific numbers a CTO can take to their next leadership meeting.

## How to Use

1. Copy this directory to `~/.claude/skills/devex-github-baseline-extraction/`
2. Install prerequisites: `gh` CLI (authenticated), `jq`, Python 3.8+, and a local git clone of the target repo (needed for stability metrics M7-M9)
3. Invoke with a trigger phrase like "extract devex baseline metrics", "collect PR cycle time metrics", "devex baseline"
4. Provide the repository (GitHub URL or local clone path) and the output directory

## Origin Story

Built as a core component of an AI coding tools adoption audit. The original problem was that the audit's "Developer Impact" section was too qualitative — it described friction in developer workflows but lacked numbers. The solution was a nine-metric (M1-M9) quantitative baseline extracted directly from GitHub PR data, generated by scripts rather than LLM output, so every number traces back to an API response or git log entry.

## Customize for Your Org

- [ ] Default extraction period is 180 days — adjust via the `since_date` parameter for shorter/longer windows
- [ ] Add custom revert/hotfix detection patterns to match your team's commit message conventions
- [ ] Adjust the parallelism setting in `collect_tier1_metrics.py` based on your GitHub API rate limit budget
